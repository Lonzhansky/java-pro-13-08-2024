
Мінуси синхронізації, коли кілька
потоків призначаються з кількох об'єктів
----------------------------------------------
Якщо у нас є кілька потоків з одного об'єкта,
то синхронізація буде обробляти перегони даних
або буде поганий/некоректний вивід.

Що станеться, якщо кілька потоків буде призначено
з кількох об'єктів? Це знову призведе до поганого
виводу чи перегонів даних.

У нас тільки 1 квиток, але через перегони даних обидва
успішно заброньовані!

Щоб вирішити цю проблему, нам потрібно застосувати
статичну синхронізацію.
