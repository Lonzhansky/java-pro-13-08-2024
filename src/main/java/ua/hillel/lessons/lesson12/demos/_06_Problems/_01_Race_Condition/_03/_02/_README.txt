
Обробка перегонів даних за допомогою синхронізації
----------------------------------------------------
Кількість квитків не може бути негативною.

Для вирішення цієї проблеми застосовуємо концепцію
синхронізації.

Синхронізація - забезпечуємо блокування об'єкта та
оголошуємо чутливу область.

Об'єкт може мати кілька потоків, але до чутливої
області може звертатися лише один потік за один раз.

Отримано 1 квиток на одного. Більше квитків немає :/
