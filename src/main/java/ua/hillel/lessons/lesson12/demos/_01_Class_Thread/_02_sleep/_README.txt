
Thread.sleep змушує поточний потік припиняти
виконання на вказаний період.

Це ефективний засіб надання процесорного часу
іншим потокам програми або іншим програмам, які
можуть працювати в комп'ютерній системі.

При виклику Thread.sleep:
- JVM попросить базову ОС надати потоковий доступ до ЦП
протягом певного часу.
- На цей час потік повністю заблокований для обробки нових
завдань. Потік повністю заблокований і не виконуватиме жодних
інших завдань, доки не завершиться сон.
- Після закінчення цього часу ОС розбудить потік (через
переривання) та надасть доступ до ЦП для відновлення з того
місця, де він зупинився.

ОДНАК:
- Thread.sleep є блокуючою операцією. Блокує потік на певну
кількість мілісекунд.
- Thread.sleep може ускладнити тестування, вплинути на поведінку
нашої програми та збити з пантелику інших розробників, яким
потрібно вникнути в наш код.
