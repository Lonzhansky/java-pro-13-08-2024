
Перегони даних у багатопоточності
-----------------------------------
Tom та Alice планували побачення та намагалися
забронювати квитки у кіно.

На жаль, в наявності був лише 1 квиток.
Tom замовив 1 квиток одночасно для них обох.

Система бронювання квитків виділила 2 потоки та
пропустила їх через метод із логікою бронювання
квитків.

Наприкінці процедури вони обидва отримали по 1-му квитку,
а кількість квитків, що залишилися, стала -1 :/
