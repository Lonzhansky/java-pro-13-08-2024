
Пріоритети потоків
---------------------
Кожен потік у Java має певний пріоритет.

У більшості випадків планувальник потоків планує потоки
відповідно до їх пріоритету (так зване попереджувальне
планування) та залежно від специфікацій JVM.

Розробник може також явно призначати пріоритети потоку
Java-програмі.

Пріоритети потоків - цілі числа, які визначають, як слід
обробляти один потік по відношенню до інших.

Діапазон допустимих пріоритетів потоків становить 1 - 10
(1 - найменший, а 10 - найвищий).

Class Thread визначає константи для деяких стандартних
пріоритетів:
MIN_PRIORITY -   1 (мінімальний)
NORM_PRIORITY -  5 (за замовчуванням)
MAX_PRIORITY -  10 (максимальний)

Пріоритет потоку вирішує, коли перемикатися з одного запущеного
потоку на інший процес називається перемиканням контексту.
Потік може добровільно звільнити управління, і потік із найвищим
пріоритетом, готовий до виконання, отримує ЦП.

Потік може бути витіснений потоком із вищим пріоритетом незалежно
від того, що робить потік із нижчим пріоритетом.

Щоразу, коли потік із вищим пріоритетом хоче запуститися,
він це робить.

Планувальник потоків використовує пріоритети під час розподілу ЦП.
Потік із найвищим пріоритетом отримає шанс на виконання першим.

Якщо два потоки мають однаковий пріоритет, то який потік отримає
шанс на виконання першим, вирішує планувальник потоків, який залежить
від постачальника JDK, тобто ми не можемо очікувати точного результату.

Пріоритет за замовчанням для основного потоку лише 5, але для всіх
інших потоків пріоритет буде успадковано з батьківського класу.
