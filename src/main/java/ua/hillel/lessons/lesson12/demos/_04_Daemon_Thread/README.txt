
Daemon Thread
-----------------

Демон-потік (Daemon Thread) в Java - потік з низьким пріоритетом,
який працює у фоновому режимі для виконання таких допоміжних завдань,
як збирання сміття (видалення з пам'яті об'єктів, що не використовуються),
видалення непотрібних записів із кешу.

Демон-потік у Java також є потоком постачальника послуг,
який надає послуги потоку користувача.
Його життя залежить від потоків-користувачів, тобто коли всі
потоки-користувачі вмирають, JVM автоматично завершує
Демон-потік.

Простіше кажучи, можна сказати, що Демон-потік надає послуги
потоку-користувачу для виконання фонових завдань підтримки.
У нього немає іншої ролі в житті, крім обслуговування
потоків-користувачів.

Особливості Демон-потоків у Java:
- Не можуть запобігти завершенню JVM, коли всі потоки-користувачі
завершують своє виконання.
- JVM завершує їх роботу, коли всі потоки-користувачі закінчують своє
виконання.
- Якщо JVM знаходить працюючий Демон-потік, то JVM завершує цей потік
і після цього вимикає його. JVM все одно, чи запущений Демон-потік
чи ні.
- Це потік із вкрай низьким пріоритетом.
