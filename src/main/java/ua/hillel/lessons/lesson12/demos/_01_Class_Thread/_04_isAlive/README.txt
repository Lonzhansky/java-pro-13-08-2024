
Метод isAlive().

По суті, цей метод внутрішньо працює дуже близько
паралельно етапам життєвого циклу потоку.

Він перевіряє, чи цей потік живий. Потік живий, якщо він був
запущений та ще не помер.

Існує перехідний період між виконанням потоку та моментом,
коли потік не виконується.

Після повернення з методу run() проходить невеликий проміжок
часу, перш ніж потік зупиниться.

Якщо ми хочемо знати, чи було викликано метод запуску класу потоку
або потік був завершений, ми повинні використовувати метод isAlive().

Цей метод використовується, щоб дізнатися, чи дійсно потік
був запущений та ще не завершився.
