
Зупинка потоку
-------------------

Зупинка потоку Java вимагає деякої підготовки коду
реалізації вашого потоку.

Клас Thread містить метод stop(), але він застарів.
Початковий метод stop() не давав жодних гарантій
щодо того, у якому стані потік було зупинено.
Це означає, що всі об'єкти Java, до яких потік мав доступ
під час виконання, залишаться у невідомому стані.
Якщо інші потоки у вашому додатку також мають доступ до
однакових об'єктів, ваш додаток може несподівано і непередбачено
завершити роботу.

Замість виклику методу stop() вам доведеться реалізувати ваш код
потоку, щоб його можна було зупинити.
