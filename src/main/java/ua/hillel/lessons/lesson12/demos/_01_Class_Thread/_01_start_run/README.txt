
Методи run() та start()
--------------------------

Ми можемо викликати метод run(), але тоді він поводитиметься
як звичайний метод, і ми не зможемо скористатися перевагами
багатопоточності.

Коли метод запуску викликається за допомогою методу start(),
тоді для виконання методу запуску виділяється новий окремий
потік, тому, якщо більше одного потоку викликає метод start(),
це означає, що їх метод запуску виконується окремими
потоками (ці потоки виконуються одночасно).

З іншого боку, якщо метод run() цих потоків викликається безпосередньо,
тоді виконання всіх їх обробляється одним і тим же поточним
потоком, і багатопоточність не матиме місця, тому вихідні
дані відображатимуть послідовне виконання потоків у вказаний
порядок.
