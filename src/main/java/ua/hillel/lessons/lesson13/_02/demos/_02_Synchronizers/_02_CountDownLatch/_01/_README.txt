
Є НАБІР ПОТОКІВ, що виконують якесь завдання,
але ХОЧЕМО, ЩОБ ці потоки запускалися тільки після
того, як основний потік завершить деяку роботу
з налаштування.

ЯК ТІЛЬКИ набір потоків запущений, ОСНОВНИЙ ПОТІК повинен
чекати та продовжувати роботу тільки після того, як набір
потоків завершить завдання.
У цьому випадку МОЖЕМО використовувати два CountDownLatch.

ПЕРШИЙ CountDownLatch використовується для запобігання
початку роботи будь-якого потоку до тих пір, поки основний
потік не завершить налаштування завдань.

ДРУГИЙ CountDownLatch діє як сигнал завершення,
який змушує набір потоків чекати завершення всіх
потоків. Це дає основному потоку сигнал для продовження.
