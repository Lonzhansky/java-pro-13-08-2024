
Lock Objects (Об'єкти блокування)
-------------------------------------

Об'єкти блокування працюють так само, як неявні блокування,
використовувані синхронізованим кодом.

Ключове слово synchronized у Java внутрішньо використовує
вбудоване блокування, пов'язане з об'єктом, для отримання
монопольного доступу до полів-членів об'єкта.

Традиційний спосіб досягти синхронізації потоків у Java
використовувати ключове слово synchronized. Хоча воно забезпечує
певну базову синхронізацію, ключове слово synchronized
досить жорстке у використанні.

Наприклад, потік може взяти блокування лише один раз.
Синхронізовані блоки не пропонують жодного механізму черги
очікування і після виходу з одного потоку будь-який потік може взяти
блокування.

Це може призвести до нестачі ресурсів для будь-якого іншого потоку
протягом дуже тривалого часу.

Замість використання вбудованого блокування за допомогою ключового
слова synchronized можемо використовувати різні класи блокування,
надані Java Concurrency API, щоб мати більш точний
контроль за механізмом блокування.
