
CyclicBarrier, що перевикористовується.

На відміну від CountDownLatch, який НЕ МОЖНА
використовувати повторно, CyclicBarrier в Java
можна використовувати повторно після звільнення
очікуваних потоків.

ТУТ, CyclicBarrier ініціалізується значенням 4,
оскільки також будемо чекати на основний потік.

Ще три потоки будуть запущені після того, як
початковий набір із трьох потоків буде звільнено
від бар'єру.

Ці нові потоки використовують один і той самий об'єкт
CyclicBarrier.
