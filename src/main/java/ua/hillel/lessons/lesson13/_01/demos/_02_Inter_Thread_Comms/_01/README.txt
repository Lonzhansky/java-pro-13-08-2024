
Коли використовувати метод notify() та notifyAll()?
-----------------------------------------------------
У разі взаємовиключного блокування лише один з очікуваних потоків
може зробити щось корисне після отримання повідомлення (у цьому
випадку отримати блокування). У такому разі краще використовувати
notify().

При правильній реалізації могли б використовувати notifyAll() і
в цій ситуації, але зайве розбудили б потоки, які однаково нічого
не можуть зробити.

У деяких випадках всі очікувані потоки можуть зробити корисні
дії після завершення очікування. Прикладом може бути набір потоків,
які очікують завершення певного завдання; як тільки завдання
завершено, всі потоки, що очікують продовжити свою роботу.

У такому випадку повинні використовувати notifyAll() для
одночасного пробудження всіх очікуваних потоків.
