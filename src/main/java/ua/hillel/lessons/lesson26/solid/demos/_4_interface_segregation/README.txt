
Принцип поділу інтерфейсу
(Interface Segregation Principle, ISP)
-------------------------------------------
Великі інтерфейси повинні розбиватися на більш дрібні.
Класи реалізації повинні використовувати лише ті методи,
які їм потрібні.
Програмні сутності не повинні залежати від методів, які
вони не використовують.
При зміні методу інтерфейсу не повинні змінюватись
програмні сутності, які цей метод не використовують.
