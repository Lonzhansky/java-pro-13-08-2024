
Принцип заміщення Лісков
(Liskov Substitution Principle, LSP)
----------------------------------------
Запроваджено Барбарою Лісков. Застосовується до
успадкування таким чином, що похідні класи мають бути
повністю замінними для своїх базових класів

Інакше кажучи, якщо клас A є підтипом класу B, ми повинні
мати можливість замінити B на A, не перериваючи поведінки
програми.

Розширює принцип відкритості/закритості, а також фокусується
на поведінці суперкласу та його підтипів.
Ми повинні проектувати класи так, щоб зберігати цю
властивість, якщо у нас немає вагомих причин чинити інакше.
