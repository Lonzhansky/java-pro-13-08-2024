
Принцип єдиної відповідальності
(Single Responsibility Principle, SRP)
---------------------------------------------
Кожен об'єкт повинен мати одну відповідальність
та ця відповідальність має бути повністю інкапсульована
в клас.
Усі його поведінки повинні бути спрямовані виключно на
забезпечення цієї відповідальності.


Демо
-------
Додаток доставки їжі, який приймає замовлення на їжу,
розраховує рахунок та веде облік доставки клієнтам.
У нас може бути 1 окремий клас для кожного з завдань,
а потім основний клас може просто викликати ці класи,
щоб виконувати ці дії одна за одною.

Клас Customer має атрибути клієнта, такі як ім'я та адресу.
Клас Order містить всю інформацію про замовлення, таку
як назва товару, кількість.
Клас BillCalculation обчислює загальний рахунок, встановлює
суму рахунку в об'єкт замовлення.
Клас Delivery має 1 завдання - доставка замовлення покупцю.
