
Indirection (Перенаправлення)
--------------------------------
Розподіляти обов'язки між об'єктами, уникаючи
прямого зв'язування. Можна надати обов'язки з
забезпечення зв'язку між компонентами чи службами
проміжному об'єкту (посереднику).

Тобто будь-який об'єкт у коді необхідно викликати
через його інтерфейс (проміжний об'єкт).

ПИТАННЯ: Як визначити відповідальність об'єкта та
уникнути сильної зв'язаності між об'єктами, навіть
якщо один клас потребує функціоналу (сервісів), який
надає інший клас?

ВІДПОВІДЬ: Необхідно наділити відповідальністю об'єкт
посередника.

ПРИКЛАД:
Шаблон MVC. UI логіці насправді потрібний не контролер,
а модель, доменна логіка. Але ми не хочемо, щоб UI логіка
була сильно пов'язана з моделлю, і можливо в UI ми хочемо
отримувати дані та працювати з різною предметною логікою.
А пов'язувати UI шар з бізнес-логікою було б неправильно,
тому що отримаємо код, який буде складний для змін та
підтримки. Вихід – створюємо Controller як посередника
між View та Model.

КОД: Див. шаблон Controller.
