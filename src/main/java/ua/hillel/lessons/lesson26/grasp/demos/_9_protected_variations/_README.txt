
Protected Variations (Стійкий до змін)
-----------------------------------------
Приховування реалізації чи захищені зміни.
ЗМІНА одних елементів системи НЕ ПОВИННА впливати
на інші.
НЕОБХІДНО ідентифікувати точки можливих змін або
нестійкості та розподілити обов'язки таким чином, щоб
забезпечити стійку роботу системи.

ПИТАННЯ:
- Як спроектувати об'єкти, щоб зміни в об'єкті або
об'єкта не торкалися інших?
- Як уникнути ситуації коли, змінюючи код об'єкта,
доведеться вносити зміни до багатьох інших об'єктів
системи?

ВІДПОВІДЬ:
Визначити "точки змін" та зафіксувати їх в абстракції
(інтерфейсі).
"Точки змін" – це наші об'єкти, які можуть змінюватись.
"Інтерфейс" - у широкому значенні, а не лише Java
interface.

ПРИКЛАД:
Точки зміни - класи EmployeeInfoA, EmployeeInfoB,
EmployeeInfoC, які надають різноманітний набір даних
про об'єкт Employee.
Інтерфейс EmployeeInfo фіксує ці точки зміни в
абстракції, тому зміна однієї точки не впливає іншу.
