
Creator (Творець)
---------------------
Сприяє НИЗЬКОЙ ЗАЛЕЖНОСТІ між класами (Low Coupling),
оскільки екземпляри класу, яким необхідно містити посилання
на деякі об'єкти повинні створювати ці об'єкти.

ЯКЩО клас створює певний об'єкт самостійно, то тим самим
він перестає бути залежним від класу, який відповідає за
створення об'єктів для нього.

СТВОРЮВАТИ екземпляри класу ПОВИНЕН клас, якому вони
потрібні.

Creator, може розглядатися як окремий випадок Information
Expert, оскільки виклик конструктора – це виклик методу.
Його дотримання разом з Information Expert дозволяє досягти
мінімальної кількості зв'язків між класами та більшої
перевикористовуваності.



Демо
------
Є найпростіша система класів: Order (замовлення), що містить
список OrderItem-ів (одиниць замовлення), елементи якого,
у свою чергу, містять Product (товар) та його кількість,
а товар може містити, наприклад, ціну, назву та ін.

Завдання: обчислити суму замовлення.
