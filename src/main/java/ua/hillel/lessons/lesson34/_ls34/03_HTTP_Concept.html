Поняття HTTP.

HTTP (Hypertext Transfer Protocol) є основою для обміну інформацією в Інтернеті.

Ключові концепції HTTP.

Клієнт-Серверна Модель. HTTP використовує модель клієнт-сервер, де клієнт (наприклад, Web-браузер) взаємодіє з сервером (наприклад, Web-сервер) для отримання ресурсів (сторінки, файли).

Запит (Request) та Відповідь (Response). Клієнт відправляє запит на сервер, а сервер відповідає на цей запит, передаючи дані або повідомлення про помилку.

URI (Uniform Resource Identifier, Уніфікований ідентифікатор ресурсу) – унікальна послідовність символів, що ідентифікує ресурс в мережі.

URI має формат:

<protocol>://<service-name>/<ResourceType>/<ResourceID>

Типами URI є URN та URL.

URN (Uniform Resource Name, Універсальне ім’я ресурсу) ідентифікує ресурс за допомогою імені, яке є одночасно унікальним та постійним. URN не завжди вказує, де знайти ресурс в Інтернеті. URN використовуються як шаблони, які використовують інші парсери для ідентифікації ресурсу.

URN слідують схемі urn і зазвичай мають префікс urn:. Наприклад, urn:isbn:1234567890 використовується для ідентифікації книги за номером ISBN (International Standard Books Number) у бібліотечному додатку.

Щоразу, коли URN ідентифікує документ, URN перетворюються на URL-адресу, після чого документ може бути завантажений.

URL (Uniform Resource Locator, Уніфікований покажчик ресурсів) містить інформацію про отримання ресурсу з його розташування.

http://abc.com/contacts.html

URL-адреси починаються з протоколу (наприклад, http) і містять інформацію про ім’я мережевого хоста (наприклад, abc.com) та шляхи до Web-документа (наприклад, /contacts.html).

URN ідентифікує ресурс, а URL вказує шлях ресурсу. URN розроблений таким чином, що ідентифікатор не залежить від того, де зберігаються дані, URL вказують на конкретне місце, де знаходиться ресурс. Обидва формати ідентифікаторів схожі один на одного за своєю конструкцією, але служать різним цілям.

Методи Запиту. HTTP визначає різні методи запиту, такі як GET (отримати ресурс), POST (надіслати дані для обробки), PUT (оновити чи створити ресурс), DELETE (видалити ресурс) та інші.

Статус-коди. Сервер повертає статус-код у відповідь, який вказує на результат обробки запиту. Наприклад, код 200 означає успішну відповідь, а 404 - що ресурс не знайдено.

Заголовки. HTTP-запити та відповіді можуть містити заголовки, які надають додаткову інформацію про запит або відповідь. Наприклад, заголовок “Content-Type” вказує на тип даних, які передаються.

Сесії та Кукі. HTTP обробляє кожен запит, який відсилається серверу, незалежно від попередніх запитів без збереження стану, але якщо стан все ж необхідно зберігати, використовуються сесії та кукі.

Безпека (HTTPS). HTTPS (Hypertext Transfer Protocol Secure) використовує шифрування для захисту конфіденційності та цілісності передачі даних між клієнтом і сервером.

Кешування. HTTP підтримує кешування, де клієнти можуть зберігати копії ресурсів для зменшення навантаження на сервер та прискорення завантаження сторінок.

Розуміння цих ключових концепцій HTTP важливо для розробників та адміністраторів мереж, оскільки це допомагає оптимізувати та усунути проблеми взаємодії між клієнтами та серверами.
