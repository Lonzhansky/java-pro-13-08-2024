
Декоратор (Decorator)
-----------------------
Дозволяє динамічно додавати об'єктам нову
функціональність, обертаючи їх у корисні «обгортки»,
не змінюючи їхню структуру.

Створює клас декоратора, який обертає вихідний
клас та надає додаткові функціональні можливості,
зберігаючи сигнатуру методів класу незмінною.

Декоратори надають гнучку альтернативу підкласам
для розширення функціональності.

Декоратор ВИКОРИСТОВУЄТЬСЯ:
- Для додавання обов'язків до окремих об'єктів
динамічно і прозоро, тобто не торкаючись інших об'єктів.
- Для додавання обов'язків, які можна відкликати.
- Коли розширення шляхом створення підкласів є недоцільним.
Іноді можлива велика кількість незалежних розширень,
що може призвести до напливу підкласів для підтримки кожної
комбінації. Або визначення класу може бути приховане чи іншим
чином недоступне для створення підкласів.


Демо
-------
Є девайс. Іноді до девайсу додається будь-який аксесуар
для надання чи посилення будь-яких властивостей.
Щоб додати аксесуар, необов'язково створювати новий девайс.
Можемо динамічно додавати до нього відповідний аксесуар.
