
Адаптер (Adapter)
-------------------
Дозволяє об'єктам з несумісними інтерфейсами
працювати разом.
Працює як міст між двома несумісними
інтерфейсів.


Використання
---------------
- Хочемо використати існуючий клас, а його
інтерфейс не відповідає тому, що необхідно.
- Хочемо створити повторно використовуваний клас,
який взаємодіє з непов'язаними або непередбачуваними
класами. Тобто класами, які необов'язково мають сумісні
інтерфейси.
- Потрібно використати кілька існуючих підкласів,
але недоцільно адаптувати їхній інтерфейс, створюючи
підкласи для всіх. Адаптер об'єкта може адаптувати
інтерфейс свого батьківського класу.

Більшість додатків, що використовують сторонні бібліотеки,
використовують адаптери як проміжний шар між додатком та
сторонньою бібліотекою, щоб відокремити додаток
від бібліотеки.

Якщо потрібно використовувати іншу бібліотеку, потрібно лише
адаптер для нової бібліотеки без зміни коду програми.


Демо
------
Машина може працювати безпосередньо з деталлю A,
але не може працювати з деталлю B.

1) Створюємо інтерфейс DetailA та клас DetailB.
2) Створюємо клас Machine.
3) Для можливості роботи машини з деталлю B,
створюємо клас-адаптер DetailBAdapter.
4) Створюємо клас Main для запуску програми.
