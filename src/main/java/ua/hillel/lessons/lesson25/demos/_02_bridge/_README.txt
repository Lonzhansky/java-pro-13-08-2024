
Міст (Bridge)
----------------
Відокремлює абстракцію від її реалізації, щоб вони
могли змінюватись незалежно один від одного.
Поділяє один або кілька класів на дві окремі ієрархії
- абстракцію та реалізацію, дозволяючи змінювати
їх незалежно один від одного.

Шаблон Bridge передбачає перевагу композиції перед
успадкуванням. Деталі реалізації передаються з ієрархії
в інший об'єкт із окремою ієрархією.


Використання
---------------
- Хочемо уникнути постійної прив'язки між абстракцією
та її реалізацією. Це може мати місце, наприклад, коли
реалізація має бути обрана або змінена під час
виконання.
- І абстракції, і їх реалізації мають бути розширеними
за рахунок створення підкласів. У цьому випадку шаблон
Bridge дозволяє комбінувати різні абстракції та реалізації
та розширювати їх незалежно.
- Зміни у реалізації абстракції не повинні впливати на
клієнтів; тобто їхній код не повинен перекомпілюватися.


Демо
------
Є девайс із різними опціями, необхідно дозволити
змішувати різні аксесуари з різними опціями.
МОЖНА створити кілька копій кожного девайсу для
кожної із опцій.
Шаблон Bridge дозволяє створити окремі опції
і встановлювати їх для девайса, за необхідності.
