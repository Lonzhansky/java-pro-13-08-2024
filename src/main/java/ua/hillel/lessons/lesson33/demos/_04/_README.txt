
Class HttpClient
=====================
Представлений у JDK 11
https://docs.oracle.com/en/java/javase/11/docs/api/java.net.http/java/net/http/HttpClient.html

HttpClient можна використовувати для надсилання запитів
та отримання відповідей.

HttpClient створюється за допомогою компонувальника (builder).
Конструктор можна використовувати для налаштування стану кожного
клієнта, наприклад: кращу версію протоколу
( HTTP/1.1 або HTTP/2 ), чи слід слідувати перенаправленням,
проксі-серверу, автентифікатору і т.д.

Після створення HttpClient є незмінним і може використовуватись
для надсилання кількох запитів.


Проблеми з HTTP-клієнтом до Java 11
--------------------------------------

ПРОБЛЕМИ HttpURLConnection:
- URLConnection API був розроблений з кількома протоколами, деякі
 нині більше не працюють.
- API передує HTTP/1.1 і надто абстрактний.
- Працює тільки в режимі блокування (тобто один потік на запит/відповідь).

На відміну від HttpURLConnection, HttpClient надає механізми
синхронних та асинхронних запитів.

API HttpClient складається з трьох основних класів:
- HttpRequest представляє запит, який буде надіслано через HttpClient.
- HttpClient веде себе як контейнер для інформації про конфігурацію, загальну
 для кількох запитів.
- HttpResponse представляє результат виклику HttpRequest.
