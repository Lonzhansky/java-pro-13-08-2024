

Linear (Sequential Search)
Лінійний (Послідовний пошук)
-------------------------------------
Цей алгоритм працює шляхом послідовного
перебору всього масиву чи списку з одного кінця
доки не буде знайдено цільовий елемент.
Якщо елемент знайдено, він повертає його індекс,
інакше -1.


Аналіз часової складності
-----------------------------
Найкращий випадок виникає, коли цільовий елемент
є першим елементом масиву.
Кількість порівнянь у такому разі дорівнює 1.
Таким чином, часова складність дорівнює O(1).

Середній випадок. Цільовий елемент буде десь
у середині масиву.
Кількість порівнянь у такому разі буде N/2.
Таким чином, часова складність буде O(N)
(Константа ігнорується).

Найгірший випадок виникає, коли цільовий елемент
є останнім елементом у масиві або не входить
у масив.
В цьому випадку нам потрібно обійти весь масив,
тому кількість порівнянь буде N.
Таким чином, часова складність буде O(N).
